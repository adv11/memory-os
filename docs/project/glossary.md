# Glossary

## MemoryOS

The product: a personal learning memory system that tracks what users learn, stores related resources, visualizes knowledge, and later supports AI-based retention.

## Topic

A broad learning area owned by a user, such as DSA, System Design, Spring Boot, or AI.

## Learning Session

A record of what a user learned on a specific day under a topic. It contains a title, description, difficulty score, resources, and concepts.

## Resource

A file or link attached to a learning session. Files are stored in the user's Google Drive. MemoryOS stores metadata and references.

## Concept

An important idea extracted or entered from a learning session, such as Class Diagram, Sequence Diagram, Aggregation, or Composition.

## Knowledge Graph

A visual representation of relationships between topics, learning sessions, and concepts. In V1, it is generated dynamically from PostgreSQL relational data.

## Revision Engine

A future system for spaced repetition and adaptive review scheduling.

## Semantic Linking

A future AI feature that discovers connections between topics and concepts.

## Knowledge Decay

A future estimate of what a user may be forgetting and what should be revised.
